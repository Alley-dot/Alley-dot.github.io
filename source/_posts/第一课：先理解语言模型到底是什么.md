## 1. 什么是语言模型

### 1.1 最朴素的定义

语言模型（Language Model）就是一个「猜下一个内容」的系统。

比如给它一句话开头：

```
今天天气很
```

它可能觉得下一个词最可能是：

- `好` (概率 0.45)
- `热` (概率 0.20)
- `冷` (概率 0.15)
- 其他...

再比如：

```
I love machine
```

它可能猜：

- `learning`
- `learning`
- `learning`

### 1.2 精确的定义

语言模型本质上是对「下一个 token 的概率分布」进行建模。

形式化地说：

```
P(next_token | previous_tokens)
```

给定前面的 token，预测下一个 token 的概率。

## 2. 什么是 Token

### 2.1 为什么说「Token」而不是「词」

这是一个非常关键的问题。

**原因一：词的数量太大**

英文有大量词形变化：

- `run`, `running`, `ran`, `runs`
- `good`, `better`, `best`

每个如果都当成独立单元，词表会爆炸。

**原因二：字符太细**

如果按字符来（`h`, `e`, `l`, `l`, `o`），序列会变得很长，模型更难学。

**原因三：新词问题**

网络时代新词不断出现：

- `ChatGPT`, `Transformer`, `SwiGLU`
- 各种梗、缩写

按词切分会遇到 OOV（Out-of-Vocabulary）问题。

### 2.2 Token 是折中方案：Subword

现代语言模型用的是 **Subword**（子词）：

```
unbelievable → ["un", "believ", "able"]
```

优点：

1. 词表可控
2. 可以处理任意新词
3. 效率与效果平衡

### 2.3 Token ≠ 词 ≠ 字

| 形式 | 例子 | 特点 |
|------|------|------|
| 字符 | `'你'`, `'A'` | 太细 |
| 词 | `"Hello"` | 太粗，可能 OOV |
| 子词 | `"un" + "believ" + "able"` | 折中 |

**重要**：文本是给人看的，Token 是给模型看的。

## 3. 从文本到预测的整条流水线

```
文本 → Tokenizer → Token IDs → Embeddings → Transformer → Logits → Softmax → 概率分布 → 预测下一个 Token
```

### 3.1 文本（Text）

人类写的原始内容：

```
The cat sat on the mat
```

### 3.2 Tokenizer

把文本切成 Token：

```
["The", " cat", " sat", " on", " the", " mat"]
```

注意：Tokenizer 可能把空格也编码进 Token 里（如 `" cat"` 带前导空格）。

### 3.3 Token IDs

把每个 Token 映射成整数：

```
"The" → 101
" cat" → 592
" sat" → 843
...
```

整句话变成整数序列：

```
[101, 592, 843, 219, 45, 1327]
```

### 3.4 Embeddings

整数没有语义，需要映射成向量：

- Token ID 是离散符号
- Embedding 是连续向量表示

Embedding table 是一个矩阵：

- 行数 = 词表大小
- 列数 = 向量维度（如 768, 1024）

查表 = 取矩阵的某一行。

### 3.5 Transformer

处理这些向量序列，结合上下文，为每个位置生成新的表示。

关键能力：让每个位置能看到「前面的所有 Token」。

### 3.6 Logits

模型输出对所有候选 Token 的「打分」：

```
对 50000 个可能 Token 的分数
```

为什么叫「Logits��？这是统计学里的术语，指「未归一化的分数」。

### 3.7 Softmax

把 Logits 变成概率分布：

```
Softmax(logits)[i] = exp(logits[i]) / sum(exp(logits[j]))
```

### 3.8 预测

选择概率最高的 Token，或按概率采样一个。

## 4. 训练时到底在学什么

### 4.1 Next-Token Prediction

训练文本：

```
I love deep learning
```

被拆成：

| 输入 | 目标 |
|------|------|
| `I` | `love` |
| `I love` | `deep` |
| `I love deep` | `learning` |

这就是 **Next-Token Prediction**（下一个 Token 预测）。

### 4.2 语言模型的目标

形式化地说：

如果 Token 序列是 `x1, x2, x3, x4`

训练目标是最大化：

```
P(x2|x1) * P(x3|x1,x2) * P(x4|x1,x2,x3)
```

也就是：

- 看到 `x1` 时猜对 `x2`
- 看到 `x1 x2` 时猜对 `x3`
- 看到 `x1 x2 x3` 时猜对 `x4`

## 5. 为什么上下文重要

### 5.1 歧义问题

```
bank
```

在不同上下文中含义不同：

- `I deposited money in the bank.` (银行)
- `We sat by the river bank.` (河岸)

只看 `bank`，模型分不清是哪个意思。

### 5.2 上下文让你分清

Transformer 的强项就是「根据上下文调整表示」。

同一个词在不同上下文里得到不同的向量表示。

## 6. Transformer 在系统中的角色

| 模块 | 职责 |
|------|------|
| Tokenizer | 把文本切成模型能处理的单位 |
| Embedding | 把离散 Token 变成向量 |
| Transformer | 在上下文中加工这些向量 |
| Output Head | 把向量变成「下一个 Token 的打分」|
| Softmax | 把打分变成概率 |

所以：

- **Tokenizer** 解决「文本怎么表示」
- **Transformer** 解决「上下文怎么计算」
- **Training** 解决「参数怎么学出来」

A1 基本上要你实现这三个关键部分。

---

## 7. 最核心的三个直觉

作为初学者，先建立这三个直觉：

### 直观一：模型处理的是 Token，不是句子本身

人类看到：

```
hello world
```

模型看到：

```
Token IDs / Embeddings / Probability Distribution
```

### 直观二：模型训练的本质是「根据前文猜后文」

不是理解宇宙，不是先天会说话。

是在大量数据上反复做：

```
给前文 → 猜下一个 Token
```

### 直观三：Transformer 的强项是处理上下文关系

它之所以强，不是因为名字高级。

而是因为它能高效地让每个位置「看见」上下文并整合信息。

## 8. 一个完整的例子

我们完整走一遍流程：

### 输入

```
I like cats
```

### 1. Tokenizer

假设切成：

```
["I", " like", " cats"]
```

### 2. Token IDs

```
[5, 27, 91]
```

### 3. 训练

模型看到 `[5, 27]`（也就是 `I like`）

要预测下一个 Token：

- 正确答案：`91`（也就是 `" cats"`）

### 4. 模型输出

对所有候选 Token 的打分：

```
" apes":  -1.2
" birds": 0.8
" bats":  1.1
" cats":  3.2
" dogs":  1.7
...
```

### 5. Softmax 后变成概率

```
" cats": 0.70
" dogs": 0.15
" birds": 0.08
...
```

### 6. Loss

因为正确答案是 `" cats"`（概率 0.70），如果模型能给更高的概率，Loss 就更低，参数更新的方向就是让「正确的下一个 Token」概率更高。

这个过程重复几亿次，模型就学会了语言中的统计规律。

## 9. 容易卡的点（预先提醒）

以后你很可能会在这些地方困惑：

1. **Token 不等于词**，也不一定等于字
2. **模型输入不是文本**，而是数字和向量
3. **语言模型输出的是概率**，不是直接一句完整的话
4. **训练和生成不是同一个��程**
5. **Transformer 里最难的不是概念名词，而是张量 shape**

后面讲到 Attention 时，我会反复用具体的 shape 来讲。

---

## 10. 本课总结

记住这 8 句话：

1. 文本不能直接喂给模型，必须先转成 Token
2. Token 是模型处理文本的基本单位
3. Token 再变成整数 ID
4. ID 再通过 Embedding 变成向量
5. Transformer 处理的是向量序列
6. 语言模型的目标是预测下一个 Token
7. 模型输出的是所有候选 Token 的概率分布
8. 训练就是不断让「正确的下一个 Token」概率变高
