## 1. 训练在做什么

你可以把训练理解成一个不断重复的循环：

```text
取一批训练数据
-> 前向计算，得到 logits
-> 和正确答案比较，得到 loss
-> 反向传播，算出每个参数的梯度
-> 用 optimizer 更新参数
-> 重复很多很多次
```

如果前面是“模型结构”，那这一课就是“模型怎么被教会”。

---

## 2. 语言模型的监督信号

### 2.1 next-token prediction

语言模型最核心的训练目标还是那一句话：

> 给前文，预测下一个 token。

假设一句 token 序列是：

```text
[BOS, I, love, cats, EOS]
```

训练时会变成：

| 输入 | 目标 |
|------|------|
| `[BOS]` | `I` |
| `[BOS, I]` | `love` |
| `[BOS, I, love]` | `cats` |
| `[BOS, I, love, cats]` | `EOS` |

所以训练数据其实是大量这种“前缀 -> 下一个 token”的样本。

### 2.2 为什么不是直接预测整句话

因为“整句话”太难，而且不方便分解优化。

而 next-token prediction 有两个巨大优点：

- 每个位置都能提供监督信号
- 可以把整个句子的概率分解成很多个局部概率的乘积

形式化地说：

```text
P(x1, x2, x3, x4)
= P(x1) P(x2 | x1) P(x3 | x1, x2) P(x4 | x1, x2, x3)
```

这就是自回归语言模型的基础。

## 3. logits 到 loss

### 3.1 logits 是什么

模型最后输出的不是概率，而是一组 raw scores：

```text
logits = [2.1, -0.3, 4.7, 0.2, ...]
```

每个值对应词表里的一个 token。

值越大，表示模型越“偏向”这个 token。

### 3.2 softmax

softmax 把 logits 变成概率分布：

```text
p_i = exp(logit_i) / sum_j exp(logit_j)
```

softmax 之后：

- 每个值都在 `[0, 1]`
- 所有值加起来等于 `1`

### 3.3 正确答案如何参与训练

假设词表大小是 5，模型输出：

```text
logits = [1.2, 0.1, 3.0, -0.4, 0.7]
```

正确 token 是第 3 个（索引 2）。

softmax 后可能得到：

```text
probs = [0.11, 0.04, 0.72, 0.02, 0.11]
```

此时模型给正确答案的概率是 `0.72`。

训练的目标就是：

> 让正确 token 的概率越来越高。

---

## 4. Cross-Entropy Loss

### 4.1 直觉理解

如果正确答案概率高，loss 应该小。
如果正确答案概率低，loss 应该大。

cross-entropy 正好满足这个直觉：

```text
loss = -log(p_correct)
```

### 4.2 数值感觉

| 正确答案概率 | loss |
|-------------|------|
| 1.0 | 0 |
| 0.5 | 0.693 |
| 0.1 | 2.303 |
| 0.01 | 4.605 |

也就是说：

- 模型越有把握且猜对，loss 越小
- 模型越离谱，loss 越大

### 4.3 在 batch 上怎么做

如果一个 batch 里有很多样本，我们通常对所有位置的 loss 取平均：

```text
batch_loss = mean(loss_1, loss_2, ..., loss_n)
```

PyTorch 里常用的是：

```python
torch.nn.functional.cross_entropy(logits, targets)
```

它内部已经把 `softmax + log + negative + mean` 都处理好了。

## 5. Perplexity

### 5.1 为什么还要看 perplexity

loss 是训练时最自然的目标，但人们也经常看 `perplexity`，因为它更像“模型有多困惑”。

定义：

```text
perplexity = exp(loss)
```

### 5.2 直觉解释

如果 perplexity = 10，可以粗略理解成：

> 模型平均相当于在 10 个候选里犹豫。

如果 perplexity = 2，说明模型已经相当确定。

### 5.3 和 loss 的关系

- `loss` 越低越好
- `perplexity` 越低越好
- 两者是一一对应的：`ppl = exp(loss)`

所以：

```text
loss = 0      -> ppl = 1
loss = 1      -> ppl ≈ 2.72
loss = 2      -> ppl ≈ 7.39
loss = 3      -> ppl ≈ 20.09
```

对于语言模型来说，`ppl=1` 意味着永远预测完全正确，基本不现实。

## 6. 反向传播在这里做了什么

### 6.1 梯度的意义

对每个参数 `w`，反向传播会告诉你：

```text
如果把 w 稍微增大一点，loss 会怎么变？
```

这个变化率就是梯度：

```text
∂loss / ∂w
```

### 6.2 梯度下降的直觉

如果某个参数的梯度是正的，说明：

- 增大它会让 loss 变大
- 所以应该把它往小调

如果梯度是负的，说明：

- 增大它会让 loss 变小
- 所以可以把它往大调

最简单的更新规则：

```text
w = w - lr * grad
```

这里 `lr` 就是 learning rate。

---

## 7. SGD 和 AdamW

### 7.1 SGD

最基础的优化器就是 SGD：

```text
w = w - lr * grad
```

优点：

- 简单
- 直观

缺点：

- 训练深模型往往不稳定
- 对学习率很敏感

### 7.2 Adam 的核心想法

Adam 不只是看当前梯度，还会维护历史信息：

- `m`: 梯度的一阶动量（类似平均方向）
- `v`: 梯度平方的二阶动量（类似平均尺度）

所以 Adam 会做到：

- 在稳定方向上走得更顺
- 对不同参数自动调整步长

### 7.3 AdamW 为什么比 Adam 更适合 Transformer

`AdamW` 里的 `W` 是 `Weight Decay`。

它和普通的 L2 惩罚很像，但实现方式更合理。

直觉上：

> 防止参数无限变大，帮助泛化，提升训练稳定性。

现代 Transformer 基本都默认用 AdamW，而不是纯 SGD。

## 8. Learning Rate

### 8.1 学习率是什么

学习率 `lr` 决定每次参数更新走多大一步。

```text
大 lr -> 学得快，但容易震荡甚至爆炸
小 lr -> 学得稳，但可能太慢
```

### 8.2 为什么不能一直固定

训练早期和训练后期的需求不同：

- 早期：需要快速找到大方向
- 后期：需要小步微调

所以常见做法是：

1. warmup
2. decay

### 8.3 warmup

训练刚开始时，参数还是随机的，梯度可能很不稳定。

如果一上来就用很大学习率，容易炸掉。

所以常见做法是：

```text
前几百 / 几千步，从很小的 lr 线性升到目标 lr
```

### 8.4 cosine decay

后期常见的是 cosine schedule：

```text
lr 从最大值平滑下降到最小值
```

曲线像半个余弦波。

CS336 A1 里也有这一部分实现。

## 9. Gradient Clipping

### 9.1 为什么要 clip

有时某一步的梯度会特别大，导致参数更新过猛，训练直接崩掉。

这叫梯度爆炸的一种表现。

### 9.2 clipping 的做法

最常见的是按全局范数裁剪：

```text
if ||g|| > max_norm:
    g = g * (max_norm / ||g||)
```

直觉就是：

> 方向保留，但把梯度整体缩小到安全范围。

### 9.3 clipping 不是“越小越好”

如果裁得太狠，训练会变慢；
如果完全不裁，训练可能不稳定。

所以它是一个稳定训练的保护措施。

## 10. 一个完整训练 step 的图景

把所有东西串起来：

```text
1. 从数据里采样一个 batch
2. 输入模型，得到 logits
3. 用 logits 和 targets 算 cross-entropy loss
4. backward() 计算所有参数的梯度
5. gradient clipping（如果需要）
6. optimizer.step() 更新参数
7. optimizer.zero_grad() 清空梯度
8. scheduler.step() 更新学习率（视实现而定）
```

在 PyTorch 里，你会很常看到这样的结构：

```python
logits = model(x)
loss = cross_entropy(logits, y)
loss.backward()
clip_grad_norm_(model.parameters(), max_norm)
optimizer.step()
optimizer.zero_grad()
```

## 12. 最容易搞错的地方

### 12.1 logits 和概率混淆

很多初学者会混淆：

- logits：模型原始输出
- softmax 后才是概率

### 12.2 target 没右移

语言模型训练里，最经典 bug 就是：

- 输入和 target 没有错开一位

### 12.3 backward / step / zero_grad 顺序错

典型顺序应该清楚：

```text
forward -> loss -> backward -> step -> zero_grad
```

### 12.4 学习率太大

loss 不下降、突然 NaN、震荡很厉害，很多时候不是模型错，而是学习率太大。

### 12.5 没有 clip 导致训练爆炸

尤其是刚开始写 Transformer 时，很容易某一步梯度爆掉。

## 13. 本课总结

你现在应该能把训练过程说成一句完整的话：

> 模型输出 logits，用 cross-entropy 和正确 next-token 比较得到 loss；再通过反向传播算梯度，用 AdamW 更新参数，并用学习率调度和梯度裁剪来保证训练稳定。

如果你能顺着这句话解释每个名词，A1 训练部分就已经入门了。
